Welcome to the **Devic Public API**.

This API lets you integrate Devic’s AI capabilities directly into your products or internal systems: create and run **agents**, orchestrate **assistants**, measure **costs**, and evaluate behaviour automatically.

The API is designed for **automation**, **backend integrations**, and SaaS platforms that want to become truly **AI-native**.

## API surface at a glance

The Devic API is divided into the following groups:

### **Agents**
- Create and list agent threads  
- Pause, resume or complete executions  
- Trigger evaluations  
- Retrieve evaluation results  
- Retrieve daily, monthly and summary costs  

### **Assistants**
- List available assistants  
- Send messages and receive responses  
- Retrieve chat histories  
- Filter conversations across assistants  

### **Documentation**
- List available docs  
- Retrieve rendered HTML or markdown  

### **Health**
- Simple availability check

## Environments and base URLs

Devic exposes multiple environments so you can separate development and production traffic.

### Production
    https://api.devic.ai

### Staging
    https://staging-api.devic.ai

Example full endpoint:

    GET https://api.devic.ai/api/v1/agents/{agentId}/threads

## Authentication

All Devic API requests use **Bearer tokens**.

Include your token in the `Authorization` header of every request:

Header format:
    
    Authorization: Bearer YOUR_API_TOKEN

Example request:

    GET https://api.devic.ai/api/v1/agents/{agentId}/threads
    Authorization: Bearer YOUR_API_TOKEN
    Content-Type: application/json

If your token is missing or invalid, the API will return **401 Unauthorized**.

## Typical flows

### 1. Running an agent

1. **Create a thread**
    
        POST /api/v1/agents/{agentId}/threads

2. **Monitor or control the execution**

        GET /api/v1/agents/threads/{threadId}
        POST /api/v1/agents/threads/{threadId}/pause
        POST /api/v1/agents/threads/{threadId}/resume
        POST /api/v1/agents/threads/{threadId}/complete

3. **Trigger or fetch evaluations**

        POST /api/v1/agents/threads/{threadId}/evaluate
        GET /api/v1/agents/threads/{threadId}/evaluations

4. **Inspect costs**

        GET /api/v1/agents/agents/{agentId}/costs/daily
        GET /api/v1/agents/agents/{agentId}/costs/monthly
        GET /api/v1/agents/agents/{agentId}/costs/summary

### 2. Conversational assistants

1. **List available assistants**

        GET /api/v1/assistants

2. **Send messages**

        POST /api/v1/assistants/{identifier}/messages

3. **Retrieve chat history**

        GET /api/v1/assistants/{identifier}/chats/{chatUid}
        GET /api/v1/assistants/{identifier}/chats
        POST /api/v1/assistants/chats

## Error handling

Most endpoints return structured errors:

    {
      "error": "ERROR_CODE",
      "message": "Human readable description",
      "details": { ... }
    }

Common status codes:
- 400 — Invalid input  
- 401 — Missing or invalid token  
- 404 — Resource not found  
- 429 — Rate limit exceeded  
- 500 — Internal server error

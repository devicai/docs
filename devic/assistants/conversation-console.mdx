title: "Consola conversacional"
description: "Descubre cómo probar, depurar y analizar el comportamiento de tus asistentes en la consola de conversación de Devic."
---

Interactua directamente con un asistente dentro de Devic para **probar su comportamiento**, **ver las herramientas que utiliza** y **realizar tareas de depuración (troubleshooting)**.

No se trata de la interfaz final del usuario o cliente —esa se gestiona desde un **endpoint API** o mediante un **widget embebido**, explicados en las secciones [API Execution](./api-execution) y [Other options](./other-options) respectivamente—.\
Esta consola está pensada para **entornos de desarrollo y ajuste fino**.

---

## Acceso a la consola

Para acceder:

1. Abre el asistente desde el panel lateral de Devic.
2. Dirígete a la pestaña **Conversations**.
3. Selecciona una conversación existente o crea una nueva.

![Acceso a la consola de conversación del asistente](/conversation-console.png)

Cada conversación muestra el **historial de mensajes** intercambiados entre el usuario y el asistente, junto con los detalles técnicos de cada interacción.

---

## Interfaz principal

La consola ofrece una vista completa de la conversación y de los eventos generados durante la misma.

![Vista principal de la consola de conversación](/conversation-console-events.png)

Desde aquí puedes:

- Enviar mensajes de prueba al asistente.
- Observar cómo responde y qué herramientas utiliza.
- Analizar en detalle los procesos internos del modelo.
- Acceder al **Chat Log** para ver los eventos del sistema.

---

## Tool Calls

Cuando el asistente ejecuta una acción (por ejemplo, consultar una base de datos, buscar en la web o enviar un correo), Devic lo muestra mediante una **Tool Call Card**.\
Estas tarjetas indican qué herramienta se ha invocado, con qué argumentos y cuál fue la respuesta.

![Ejemplo de tool calls ejecutadas por el asistente](/tool-call-assistant-detail.png)

Puedes expandir cada tarjeta para ver la información completa del evento, incluyendo los parámetros enviados y los resultados obtenidos.

---

## Logs y depuración

La pestaña **Logs** ofrece una vista detallada de todos los eventos registrados durante la conversación, incluyendo las llamadas a la API del modelo, tool calls, respuestas parciales y errores.

![Vista del Chat Log con seguimiento de tool calls](/acces-logs-view-conversation-console.png)

## 

![Vista del Chat Log con seguimiento de tool calls](/logs-assistant.png)

Desde esta vista puedes:

- Filtrar por tipo de evento (`LOG`, `WARN`, `ERROR`, `DEBUG`).
- Seguir el orden cronológico de ejecución.
- Analizar tiempos de respuesta, mensajes internos y contenido procesado.
- Visualizar los parámetros exactos de cada **tool call**.

Esto permite detectar fácilmente:

- Por qué una tool no devolvió el resultado esperado.
- Si el asistente está interpretando correctamente el prompt.
- Cuándo interviene el modelo LLM y cuándo una herramienta externa.

---

## Análisis de mensajes

Puedes inspeccionar cada mensaje intercambiado (tanto del usuario como del asistente) para revisar su estructura JSON completa, incluyendo entradas, respuestas y metadatos.

![Detalle del mensaje con respuesta de una tool](/log-assistant-detail.png)

Esta función resulta esencial para comprender cómo el asistente:

- Procesa el contexto recibido.
- Decide qué herramienta usar.
- Genera la respuesta final visible en la conversación.

---

## Buenas prácticas para troubleshooting

- **Revisa los Logs** tras cada prueba para identificar posibles errores en tool calls o en la configuración de MCPs.
- **Valida los argumentos** enviados a cada herramienta antes de publicarlo en producción.
- **Utiliza conversaciones cortas y directas** durante las pruebas iniciales.
- **Evalúa el rendimiento**: revisa tiempos de respuesta y uso de tokens en cada ejecución.
